Observações:

1) Os campos alfanuméricos deverão ser alinhados a esquerda e totalmente preenchidos.
   Caso o campo não seja usado, deverá ser preenchido com espaços em branco.
2) Os campos numéricos serão sempre alinhados a direita. Quando houver casas
   decimais, deverão ser sempre preenchidos e usados o ponto (.) como separador
   decimal. Deverão ser totalmente preenchidos. Ex: Se o valor é 1 e o tamanho do campo
   é 6 posições, deverá ser preenchido com 000001. Caso o campo não seja usado,
   deverá ser preenchido com zeros.
3) Os campos de data serão alfanuméricos e sempre no formato DD/MM/AAAA. Os
   campo de hora serão alfanuméricos e sempre no formato HH:MM:SS. Caso o campo
   não seja usado, deverá ser preenchido com espaços em branco.
4) O diretório de arquivos padrão onde deverão ser lidos e gerados os arquivos TXT é
   c:\Inova\ReaderManager\Interface.
5) O termo ‘exportação’ refere-se a transferência de dados do Software de Gestão para o
   Coletor (PC => Coletor) e o termo ‘importação’ refere-se a transferência de dados do
   Coletor para o Software de Gestão (Coletor => PC).
6) Deverá obedecer a ordem de Tipo do Registro(A01, A02 . . .).
7) Quando os campos de CÓDIGO DO CONSUMIDOR dos registros A11 em diante
   tiverem apenas conteúdo numérico devem sempre ser preenchidos com zeros a
   esquerda de forma que todos os campos CÓDIGO DO CONSUMIDOR sempre fique
   com o mesmo número de posições. Isto é necessário uma vez que o campo no layout é
   alfanumérico.
8) Coluna Default tem a função de orientar o que deve ser enviado em um determinado
   campo caso não tenha o mesmo no Software de Gestão.

                                                          Envio

                                           Tipo de Processo 01

Exportação READER Mobile

Nome do Arquivo:

Tipo 10 => AAMMDDLL.REM (AA = Ano, MM = Mês, DD = Dia e LL* = Nr. Leiturista)
Tipo 11 => AAMMEELL.REM (AA = Ano, MM = Mês, EE* = Área e LL* = Nr. Leiturista)

*O LL pode ser usado para outra informação, como a Rota, Setor, dentre outras conforme a
necessidade.

*Caso a Rota, Setor, outra informação tenha mais de duas posições, pode ser usado o tipo 11 sendo
que irá suportar até 4 posições (EELL) para esta informação.

                                           Tipo de Processo 02

Importação Grupo de Leitura

Nome do Arquivo:
AAMMGGGG.REM (AA = Ano, MM = Mês, GGGG = Grupo de Leitura)

Nome do Arquivo Compactado:
AAMMGGGG.REM.ZIP (AA = Ano, MM = Mês, GGGG = Grupo de Leitura)

                                                   Importação Fotos
       Para fazer a importação das fotos deve-se verificar os campos existe foto no registro de
consumidores, se os mesmos estiverem sido preenchidos com ‘S’ será preciso verificar na pasta
c:\Inova\ReaderManager\Fotos a existência das fotos para que possa importá-las. Existem 4 tipos de foto:
01-Foto hidrômetro, 02-Foto fachada, 03-Foto prova adulteração hidrômetro, 09-Foto outros.
Os nomes das fotos serão preenchidos da seguinte maneira:

        Quando não utiliza mês/ano de referência no nome da foto:
         AA_BBBBBBBBBBBBBBB_CC, onde AA=Tipo da foto, BBBBBBBBBBBBBBB=Código do
